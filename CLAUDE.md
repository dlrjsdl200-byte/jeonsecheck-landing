# 계약체크 (ContractCheck) — AI 계약서 체크 포인트 정리 도구

## 프로젝트 개요
각종 계약서를 사진/PDF로 업로드 → AI 30초 체크 → 쉬운 말 해설 + 참고 대화 예시 + 추천 특약 문구 제공.

**서비스 성격**: 법률 자문·평가·분석 서비스가 아닌, 서류 항목을 자동으로 확인해주는 **기술적 도구**
**첫 번째 서비스**: 전세/월세 계약 체크 (계약서 + 등기부등본 + 건축물대장)
**향후 확장**: 근로계약서, 프랜차이즈, 중고차, 인테리어 계약 등

**슬로건**: "도장 찍기 전 30초, 전세 사고 예방"
**타겟**: 23~32세 사회초년생, 신혼부부, 지방→수도권 직장인

## 기술 스택
| 영역 | 기술 | 비고 |
|------|------|------|
| Frontend | Flutter (Web+Mobile) | 크로스플랫폼 |
| Backend | Supabase (Auth, DB, Storage, Edge Functions) | PostgreSQL 기반 |
| AI 엔진 | Claude Haiku 4.5 → Sonnet 4.6 (fallback) | 프롬프트 캐싱 90% 절약 |
| 결제 (Phase 0) | 카카오페이 송금 + 계좌이체 | PG사 없이 운영, 구글시트 수집 |
| 결제 (Phase 2+) | PG사 미정 | 매출 안정 후 전환 |
| 알림 | 카카오 알림톡 (Solapi 중계) | 전화번호 기반 |

## 아키텍처
```
Flutter App → Supabase Auth (카카오 로그인)
           → Supabase Storage (문서 업로드, 확인 후 즉시 삭제)
           → Supabase Edge Function → Claude API (OCR+항목확인+교차대조)
           → Supabase DB (결과 암호화 저장)
           → 카카오페이 송금/계좌이체 (Phase 0, PG 없이)
           → 카카오 알림톡 (알림)
```

## API 호출 구조
- **1회차**: 문서 OCR + 개별 항목 확인 (계약서/등기부/건축물대장)
- **2회차**: 교차 대조 + 확인 결과 리포트 생성
- 건당 비용: Haiku ~64원 / Sonnet ~193원

## AI 체크 핵심 항목
1. **계약서**: 표준계약서 판별 (6개 중 4개+), 특약사항, 계약기간/보증금/갱신요구권
2. **등기부등본**: 소유권(갑구), 근저당(을구), 압류/가압류/경매/처분금지/신탁(all or nothing 감점), 깡통전세 참고 수치 (HUG 기준)
3. **건축물대장**: 용도, 위반건축물, 무허가, 면적
4. **교차검증**: 소유자 3중 대조, 주소 3중 대조, 면적/용도
5. **생성**: 체크 항목 충족률, 항목별 사실 요약, 참고 대화 예시, 추천 특약 예시, 체크리스트, PDF 리포트

## 법적 준수 가이드라인 (최우선 준수사항)

### 법적 배경
- **변호사법 제109조 제1호**: 비변호사의 법률사무 취급 일반 금지
- **대법원 판례 (2026.03)**: 로폼 법률문서 자동작성은 합법 판결. 단, 생성형 AI가 구체적 사건에 법적 추론/평가를 하면 법률사무 취급에 해당할 수 있다고 경고
- **핵심 기준**: "구체적·개별적 사실관계 파악 + 법규 내용 검토 + 법적 추론/평가"가 가미되면 법률사무

### 계약체크의 포지셔닝: "서류 체크 포인트 정리 도구" (기술적 도구)
법률 자문·평가·분석(X) → 서류 항목 자동 확인 + 공식 기준 참고 정보 제공(O)
이 서비스는 계약을 평가하거나 법적 분석을 하는 것이 아니라, 서류의 항목을 기계적으로 대조·확인해주는 **기술적 도구**입니다.
- 표준계약서 항목 포함 여부 → 서식 검토 (OK)
- 등기부등본 근저당 금액 추출 → 사실 정보 추출 (OK)
- 건축물대장 용도 확인 → 사실 확인 (OK)
- 3서류 교차 검증 (소유자/주소 일치) → 사실 대조 (OK)
- HUG 공식 기준 참고 안내 → 출처 명시한 정보 제공 (OK)
- 특약 포함 여부 확인 → 법률 정보 제공 수준 (OK)

### AI 응답 원칙 — 절대 금지 표현
| 금지 | 이유 |
|------|------|
| "이 계약은 하지 마세요" | 판단/결론 = 법률사무 |
| "이 계약은 위법입니다" | 법적 판단 |
| "해지하세요", "고소하세요" | 법적 행동 지시 |
| "이 조항은 무효입니다" | 법적 효력 판단 |
| "임대인이 사기를 치고 있습니다" | 상대방에 대한 법적 판단 |
| "보증보험 가입 불가 수준" | 가입 가능 여부 단정 |
| "주임법 적용 불가" | 법 적용 여부 단정 |
| "위험합니다" (단정형) | 법적 평가에 해당 |

### AI 응답 원칙 — 안전한 표현
| 안전 표현 | 유형 |
|-----------|------|
| "일반적인 계약과 다른 점이 있습니다" | 사실 비교 |
| "주의가 필요할 수 있습니다" | 주의 환기 |
| "전문가(변호사/법무사) 상담을 권장합니다" | 전문가 연결 |
| "HUG 기준 참고 시 보증보험 가입이 어려울 수 있는 수준입니다" | 공식 기준 인용 + 완화 |
| "주택임대차보호법 보호를 받기 어려울 수 있어요. 전문가 확인을 권합니다" | 가능성 + 전문가 연결 |
| "표준계약서에는 포함되어 있으나 이 계약서에는 누락되어 있습니다" | 항목 비교 |

### 면책 문구 — 3곳에 반드시 노출
1. **히어로 섹션 바로 아래**: "본 서비스는 법률 자문이나 계약 평가가 아닌, 서류 항목을 자동 확인해주는 기술적 도구입니다. 최종 판단은 전문가와 상의하세요."
2. **체크 결과 화면 상단**: "서류 항목 자동 확인 결과이며, 법률 자문이나 계약 평가가 아닙니다"
3. **푸터**: 전체 면책 조항

### 모든 체크 결과에 포함해야 하는 문구
```
본 결과는 서류 항목을 자동으로 확인한 참고 정보이며, 법률 자문이나 계약 평가가 아닙니다.
이 도구는 계약의 안전 여부를 판단하지 않습니다.
실제 계약 전 반드시 전문가(변호사, 법무사, 공인중개사)와 상담하시기 바랍니다.
```

### AI 시스템 프롬프트에 반드시 포함할 지시
```
당신은 전세/월세 계약 서류 항목을 자동으로 확인해주는 기술적 도구입니다.
당신은 계약을 평가하거나 법적 분석을 하는 것이 아닙니다.
서류의 항목을 기계적으로 대조·확인하고, 공식 기준을 참고하여 정보를 정리합니다.
절대 법적 판단, 계약 체결/해지 권유, 법적 조언을 하지 마세요.
"~하세요", "~하지 마세요" 같은 지시형 표현을 사용하지 마세요.
"위험합니다", "불가합니다" 같은 단정형 표현을 사용하지 마세요.
항상 "~일 수 있습니다", "~확인이 필요합니다", "~전문가 상담을 권장합니다"로 표현하세요.
공식 기준(HUG, 주임법 등)을 인용할 때는 반드시 출처를 명시하세요.
특약 문구 제안 시 "일반적으로 사용되는 예시"임을 밝히고, 전문가 상의를 안내하세요.
점수는 "체크 항목 충족률"이며, 계약의 안전도 점수가 아닙니다.
요약은 "확인한 항목 요약입니다"로 시작하고, 사실 나열만 합니다.
```
